## 0. 范围定位（冻结）

1. 今天的唯一主目标是：把 Day 5 的任务列表能力和 Day 9 的成本记录能力接到一个最小前端首页上。
2. 今天优先解决的问题是：系统已经有状态、运行和成本数据，但还没有一个可视化入口帮助用户快速查看。
3. 今天明确不做：复杂路由系统、登录鉴权、详情抽屉、日志流式订阅、复杂筛选和编辑能力。

## 1. 边界决策（已冻结）
1. 前端按既定技术栈使用 `React + TypeScript + Vite + Tailwind CSS + TanStack Query`：**YES**
