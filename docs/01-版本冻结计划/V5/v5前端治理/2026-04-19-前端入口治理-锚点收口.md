## 上一棒缺口

- 上一棒只完成了“顶部入口可见 + 页内锚点收口”。
- `交付物中心`、`审批收件箱` 虽然已有入口，但实际仍是当前长页内锚点，不是真正跨页可达。
- 所谓“第三个入口”并不是独立路由页面，而是 `project-detail` 页内区块；如果不写清，入口语义会与实际跳达模型不一致。

## 本轮修复

1. 将顶部统一导航拆分为两类：
   - 页内 section：`project-detail`
   - 跨页入口：`deliverable-center`、`approval-inbox`
2. 在 `ProjectOverviewPage.tsx` 内补充轻量页面态与 hash 导航协议，避免把本轮扩大成正式路由重构。
3. 点击跨页入口时，先切换到目标页面态，再执行目标 DOM 落点定位。
4. 对目标 DOM 挂载时机晚于滚动触发的情况补充重试滚动，降低“点得见但落不准”的风险。
5. 移除 `App.tsx` 中对 `deliverable-center / project-detail` 的抢跑滚动，统一由页面内部导航状态驱动跳达。

## 实际验证结果

- `project-detail` 保持为 `ProjectOverviewPage` 内部 section。
- `deliverable-center`、`approval-inbox` 已改为页面态切换后再定位，不再依赖当前长页锚点滚动。
- 已补充 hash 解析与挂载后重试滚动，覆盖“目标 DOM 渲染晚于滚动触发”的最小闭环。
- 所谓“第三个入口”当前仍是 `project-detail` 页内区块，不是独立路由页；本轮已按该事实处理并在文档中明确说明。
- `apps/web` 已执行 `npm.cmd run build`。
- 构建结果以本轮实际执行输出为准。

- 本轮不处理 `RoleCatalogPage / SkillRegistryPage / RoleWorkbenchPage` 等其它区域的入口重构。
