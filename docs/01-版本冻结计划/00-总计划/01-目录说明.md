
# 目录说明

## 目录规则

1. 总计划只放总览和目录说明。
2. V1 / V2 / V3 各自独立。
3. 版本内部按模块拆分。
4. 每个模块固定分成：
   - `01-计划文档`
   - `02-测试验证`
5. 天级文件命名规则：
   - 计划文档：`DayXX-主题.md`
   - 测试文档：`DayXX-主题-测试.md`

## 使用方式

1. 以后新增一天时，先补对应模块下的计划文档。
2. 同一天必须同步补测试验证文档。
3. 已完成项要写证据，进行中项要写缺口，未开始项不伪造完成记录。
4. V3 暂不创建天级计划，只保留结构。

## 正式文档 / 本地草稿边界

1. `docs/01-版本冻结计划/` 只放已确认的正式文档。
2. 临时想法、半成品规划、历史散落文档，统一放到 `.tmp/local-drafts/` 本地保留。
3. 本地草稿确认后，再迁移进入正式目录，不直接把未确认内容推到 GitHub。
4. 本次重构前的旧版阶段文档与每日计划，已迁出正式入口，并本地归档到 `.tmp/local-drafts/legacy-docs/`。
