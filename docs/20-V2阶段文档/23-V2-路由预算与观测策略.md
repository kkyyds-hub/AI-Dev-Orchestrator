## 4. Day 6 路由打分维度与命名（冻结）

### 4.1 稳定分项字段

| 字段 code | 标签 label | 分值方向 | 口径说明 |
| --- | --- | --- | --- |
| `priority` | 优先级 | 正向加分 | `low=+100 / normal=+200 / high=+300 / urgent=+400` |
| `risk_level` | 风险等级 | 正负调整 | `low=+20 / normal=0 / high=-20` |
| `human_status` | 人工状态 | 正向微调 | `resolved=+10`，其他默认 `0` |
| `execution_attempts_penalty` | 执行次数惩罚 | 负向扣分 | `-20 * execution_attempts` |
| `recent_failure_penalty` | 近期失败惩罚 | 负向扣分 | `-35 * recent_failure_count` |
| `readiness_gate` | 就绪检查 | 门禁说明 | 不可执行时仅输出阻塞说明，不给总分 |

### 4.2 总分公式

1. 就绪任务：`routing_score = sum(all score items)`。
2. 不就绪任务：`routing_score = null`，并通过 `readiness_gate` 返回阻塞原因。
3. 排序规则：先按 `routing_score` 降序，再按 `created_at` 升序，最后按标题字典序。

### 4.3 解释字段命名规范

1. `route_reason`：稳定文本摘要，供列表/历史快速阅读。
2. `routing_score`：总分，供排序和简要展示。
3. `routing_score_breakdown`：分项数组，结构为 `{code, label, score, detail}`，供日志、回放和详情页复用。
