## 2. P9 范围定义

### 2.1 P9 是什么

P9 目标是 **controlled runtime / executor orchestration** — 在安全 gate 全过的前提下，受控启动和管理外部执行器。

P9 回答的问题：
- 如何在不暴露 secret / 不绕过审批的前提下，受控启动 Codex / Claude Code / DeepSeek？
- 如何追踪执行器 session 生命周期（spawn → running → idle → done → killed）？
- 如何检测执行器 activity 并回填 cost / token usage？
- 如何在执行器活动期间保持 workspace 绑定安全？
- 如何提供用户中断 / 超时 / 清理能力？

### 2.2 P9 不是什么

| # | 不是 | 说明 |
|---|------|------|
| 1 | **不是**直接启动 Codex / Claude Code / DeepSeek | 必须先有 domain contract、safety gate、approval gate、workspace gate、cost/budget gate |
| 2 | **不是**产品运行时 Git 写闭环 | P9 initial runtime 不触发 git add/commit/push/PR/merge |
| 3 | **不是**自动 PR / merge | 产品运行时 Git 写是独立的安全门，不与 P9 runtime 捆绑 |
| 4 | **不是**AI 主管偷偷启动执行器 | 必须先 feature flag + human confirmation |
| 5 | **不是**无限并发 / 无限预算 | 必须有 concurrency limit + cost/budget guard |
| 6 | **不是**无超时运行 | 必须可配置 timeout + kill switch |

### 2.3 P9 必须先建立的硬 Gate（在真实启动之前）

| # | Gate | 说明 |
|---|------|------|
| 1 | **Feature Flag Gate** | feature flag default off，永不为 True 除非用户显式开启 |
| 2 | **Human Confirmation Gate** | 用户显式确认后才允许进入 runtime |
| 3 | **Executor Readiness Gate** | executor status == AVAILABLE，launch preview ready == True |
| 4 | **Workspace/Worktree Gate** | workspace 必须存在、合法、已绑定，不能任意目录启动 |
| 5 | **Cost/Budget Gate** | 必须在 daily/session budget 范围内，超预算拒绝启动 |
| 6 | **Concurrency Gate** | 同时运行的 executor session 不能超过上限 |
| 7 | **No Secret Exposure Gate** | 任何 response/log/event 中不得出现 API key/token/secret 明文 |
| 8 | **No Product Runtime Git Write Gate** | gate_allows_write 永不为 True |
| 9 | **Event/Audit Gate** | 所有 runtime 事件必须 append-only 记录 |
| 10 | **Timeout/Cancellation Gate** | 每个 session 必须有可配置 timeout，必须可被用户取消 |
| 11 | **Cleanup Gate** | session 结束后必须清理 workspace 绑定 |

## 5. P9 安全模型草案

### 5.1 安全原则总表

| # | 原则 | 规则 |
|---|------|------|
| 1 | **Feature Flag Default Off** | `EXECUTOR_RUNTIME_ENABLED` 默认 False，永不为 True 除非用户显式开启 |
| 2 | **User Explicit Confirmation** | 每次 executor launch 前必须用户显式确认，确认 scope 明确 |
| 3 | **Executor Readiness Must Be Available** | executor status == AVAILABLE + launch preview ready == True |
| 4 | **Launch Preview Must Exist Before Runtime** | 每次 runtime launch 前必须先生成 launch preview |
| 5 | **Workspace/Worktree Must Be Explicit and Safe** | 必须绑定合法 workspace，不能任意目录启动 |
| 6 | **No Secret Exposure** | 任何 response/log/event/DTO 中无 API key/token/secret 明文 |
| 7 | **No Env Dump** | 不 dump 环境变量到任何输出 |
| 8 | **No Native Config Path Dump** | 不在 response 中暴露原生配置文件路径 |
| 9 | **Timeout / Cancellation / Kill Switch** | 每个 session 必须有可配置 timeout，必须可被用户取消 |
| 10 | **Concurrency Limit** | 同时运行的 executor session 不超过可配置上限 |
| 11 | **Cost/Budget Guard** | 必须在 daily/session budget 范围内 |
| 12 | **Event/Audit Append-Only** | 所有 runtime 事件必须记录，不可篡改 |
| 13 | **Human Approval Before Any Product Runtime Git Write** | Git 写操作独立审批，不与 executor launch 捆绑 |
| 14 | **Product Runtime Git Write Remains Separate** | P9 initial runtime 不触发 Git 写 |

### 5.2 Feature Flag 设计

```text
EXECUTOR_RUNTIME_ENABLED: bool = False  # 默认关闭
EXECUTOR_RUNTIME_MAX_CONCURRENT: int = 1  # 默认最多 1 个并发
EXECUTOR_RUNTIME_DEFAULT_TIMEOUT_SECONDS: int = 600  # 默认 10 分钟
EXECUTOR_RUNTIME_MAX_TIMEOUT_SECONDS: int = 3600  # 最大 1 小时
EXECUTOR_RUNTIME_ALLOWED_EXECUTORS: list[str] = []  # 空白名单 = 全禁止
EXECUTOR_RUNTIME_ALLOWED_WORKSPACE_ROOTS: list[str] = []  # 空白名单 = 全禁止
```

### 5.3 Launch 安全流程

```text
UserAction → DirectorActionProposal（P7）
  → LaunchPreview 生成（P8-E，复用）
  → HumanConfirmation（用户显式确认）
  → FeatureFlagGate（EXECUTOR_RUNTIME_ENABLED == True）
  → ExecutorReadinessGate（status == AVAILABLE）
  → WorkspaceGate（workspace 合法、存在、未占用）
  → BudgetGate（daily/session budget 未超）
  → ConcurrencyGate（未超并发上限）
  → RuntimeSafetyGate（全部 gates passed）
  → ControlledRuntimeService.launch（受控启动）
  → EventStream（emit executor.session.spawned）
  → ActivityDetection（轮询进程 + JSONL）
  → Cost/Token Readback（从原生 JSONL 回填）
  → Completion / Timeout / UserCancellation
  → Cleanup（解绑 workspace，清理临时文件）
  → EventStream（emit executor.session.completed）
```

### 5.4 安全边界红线

| # | 红线 | 后果 |
|---|------|------|
| 1 | 不能跳过 human confirmation 直接启动 | 测试/验证阶段也必须 mock confirmation |
| 2 | 不能在非 workspace 目录启动 executor | WorkspaceGate 必须拒绝 |
| 3 | 不能暴露 API key / token / secret | 任何代码路径都不能 |
| 4 | 不能超过并发上限 | ConcurrencyGate 必须拒绝 |
| 5 | 不能超预算启动 | BudgetGate 必须拒绝 |
| 6 | 不能不记录 audit event | 每个 runtime 事件必须 append-only |
| 7 | 不能没有 timeout | 每个 session 必须有超时配置 |
| 8 | 不能没有 kill switch | 用户必须可以取消运行中的 session |
| 9 | 不能自动触发 Git 写 | 产品运行时 Git 写是独立 gate |
