## 2. 阻塞原因

当前 P9 虽然已完成 scoped closure（P9-A～Final），但存在一个核心阻塞：

| 阻塞点 | 说明 |
|--------|------|
| **仅有 FakeExecutorAdapter** | `controlled_runtime_service.py` 中的 `FakeExecutorAdapter` 只更新 in-memory domain snapshot，不启动任何外部进程 |
| **无 RealExecutorAdapter** | 没有真正的 subprocess 管理层来启动 Codex / Claude Code / DeepSeek CLI |
| **无真实进程管理** | 没有 process_id 追踪、没有 stdout/stderr capture、没有 signal kill |
| **无真实 activity detection** | 没有从原生 JSONL / terminal output 检测执行器活动状态的能力 |
| **无真实 cost/token readback** | 没有从原生 session file 回填 token usage 的能力 |

因此：
- P9 scoped closure 虽然 Pass，但只到 **fake dry-run evidence**。
- **P9 real executor launch 仍 Not started**。
- P9-RGWP-Pilot（真实 Git 写试点）缺少"真实执行器启动"这个前置条件。
