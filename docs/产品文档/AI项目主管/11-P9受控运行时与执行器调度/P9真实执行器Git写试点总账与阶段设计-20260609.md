## 2. 试点范围定义

### 2.1 本试点是什么

本试点属于 **P9 real executor launch 的高风险后续轨道**，目标是让外部执行器（Codex / Claude Code / DeepSeek）在严格 gate 下执行一次**受控、可回滚、可审计的最小 Git 写**。

### 2.2 本试点不是什么

| # | 不是 | 说明 |
|---|------|------|
| 1 | **不是**开启自由写权限 | 仅限一次一条明确的试点操作 |
| 2 | **不是** main 直写 | main/master 一律禁止 |
| 3 | **不是**自动 PR / merge | PR 创建和 merge 决策必须人工触发 |
| 4 | **不是**默认打开真实写能力 | 所有 feature flag 默认关闭 |
| 5 | **不是** CI 自动触发 | 试点提交不应触发 CI pipeline |
| 6 | **不是**对已有安全门的降级 | P4/P8/P9/GitWrite 所有 gate 必须通过 |
| 7 | **不是**真实执行器接管 Git | 执行器只写试点分支上的试点文件 |
| 8 | **不是**无限预算 | 必须在 budget gate 内 |

### 2.3 试点最小范围

```text
Pilot scope = ONE executor
            + ONE branch（ai/gitwrite-pilot/<date>-doc-only）
            + ONE file type（docs/*.md only）
            + ONE commit
            + NO push（unless user explicitly enables for this pilot）
            + NO PR
            + NO merge
            + NO force
```

## 4. 强制 Gate 清单

以下 17 个 gate 构成 P9 真实执行器 Git 写试点的完整准入门槛。每个 gate 必须显式通过，任一失败 = **Blocked**。

| # | Gate | 目的 | 不满足时 | P9/GitWrite 前置状态 | Pilot 前需补齐 |
|---|------|------|---------|--------------------|-------------|
| 1 | **Feature Flag Gate** | 三个 flag 必须同时为 True | **Blocked** | P9-C RuntimeFeatureFlagPolicy + GitWrite-B 已有 domain 定义 | 需要真实 feature flag 配置源 |
| 2 | **Executor Readiness Gate** | executor status == AVAILABLE | **Blocked** | P8-C discovery service + P8-D readiness API | 需要真实 probe（非 noop） |
| 3 | **Workspace/Worktree Gate** | workspace 必须存在、合法、已绑定 | **Blocked** | P9-B ExecutorRuntimeWorkspaceBinding + P4 RepositoryWorkspace | 需要真实 worktree 创建验证 |
| 4 | **Target Branch Allowlist Gate** | 仅允许白名单分支 | **Blocked** | GitWrite-B GitWriteSafetyGate 已有 TARGET_BRANCH_ALLOWLIST | 需要真实 branch allowlist 配置 |
| 5 | **Diff Preview Gate** | 写前必须展示完整 diff | **Blocked** | GitWrite-C preview service + P4 Git Diff Dry-run | GitWrite-C 已有 |
| 6 | **Secret Scan Gate** | diff 中不得含疑似 secret | **Blocked** | GitWrite-C SecretScanner + domain _SUSPICIOUS_TEXT_PATTERN | GitWrite-C 已有 |
| 7 | **Human Approval Gate** | 用户当轮独立确认 | **Blocked** | GitWrite-B GitWriteApproval domain + P4 HumanApprovalGate | 需要真实 UI 确认入口 |
| 8 | **One-Shot Token Gate** | 一次性 token，用完即废 | **Blocked** | GitWrite-B OneShotApprovalToken domain | 需要真实 token 生成器 |
| 9 | **Budget/Cost Gate** | 在 daily/session budget 范围内 | **Blocked** | P9-C RuntimeBudgetGateInput + BudgetGuardService | 需要真实 cost estimation |
| 10 | **Timeout/Kill Switch Gate** | session 必须可超时、可终止 | **Blocked** | P9-D timeout/cancel/kill + P9-C timeout_configured | P9-D 已有（fake） |
| 11 | **Rollback Plan Gate** | 写前必须准备回滚方案 | **Blocked** | GitWrite-C RollbackPlanGenerator | GitWrite-C 已有 |
| 12 | **No Direct Main Write Gate** | 禁止写 main/master | **Blocked** | GitWrite-B target_branch validation（排除 HEAD） | GitWrite-B 已有 |
| 13 | **No Force Push Gate** | 禁止 --force push | **Blocked** | GitWrite-C ForcePushDetector | GitWrite-C 已有 |
| 14 | **No Auto PR / Merge Gate** | 禁止自动创建 PR 或 merge | **Blocked** | GitWrite operation_kinds 不含 merge_pr | GitWrite-B 已有 |
| 15 | **Append-Only Audit Gate** | 每一步记录不可篡改 audit event | **Blocked** | GitWrite-B GitWriteAuditEvent + P9-B RuntimeEvent | 需要持久化 event storage |
| 16 | **Post-Write Verify Gate** | 写后验证 commit 存在、diff 匹配 | **Blocked** | 不适用（P9-RGWP 新增） | 需设计 |
| 17 | **Manual Final Confirmation Gate** | 写后人工检查通过才完成试点 | **Blocked** | 不适用（P9-RGWP 新增） | 需设计 |

## 7. 回滚策略

### 7.1 写前必须记录

| 必须记录 | 说明 |
|---------|------|
| `base_commit` | 试点写之前的 HEAD commit SHA |
| `target_branch` | 试点写入的目标分支全名 |
| `rollback_plan_id` | 回滚方案唯一 ID（GitWrite-C 已有 contract） |
| `restore_branch_hint` | 回滚时恢复到哪个分支状态 |
| `restore_commit_hint` | 回滚时恢复到哪个 commit |

### 7.2 允许的回滚操作（第一轮试点）

| # | 允许 | 禁止 |
|---|------|------|
| 1 | `git revert <pilot-commit>`（创建 revert commit） | `git reset --hard` |
| 2 | 删除未合并的试点分支（`git branch -D <pilot-branch>`） | `git push --force` |
| 3 | 人工手动回滚 | 自动回滚脚本 |

### 7.3 回滚验证

- 回滚后必须确认 base_commit 状态恢复。
- 回滚操作也必须记录 audit event。
- 回滚后试点分支可以保留用于参考，但不应留在远程。

## 8. 用户显式确认策略

### 8.1 确认原则

| # | 原则 | 说明 |
|---|------|------|
| 1 | **当轮独立确认** | 每次试点需要用户当轮显式确认，不能复用历史确认 |
| 2 | **完整信息展示** | 确认前必须展示所有关键信息 |
| 3 | **风险明确告知** | 确认界面必须告知试点风险 |
| 4 | **One-shot token** | 确认后生成一次性 token，用完即废 |
| 5 | **限时有效** | token 有效期 5 分钟，过期需重新确认 |

### 8.2 确认界面必须展示的内容

| 展示项 | 说明 |
|--------|------|
| executor | 哪个执行器执行试点（如 Codex） |
| branch | 目标试点分支 |
| file scope | 将修改的文件列表 |
| operation | 将要执行的操作（create_commit） |
| diff preview | 将要写入的 diff 摘要 |
| rollback plan | 回滚方案 |
| safety statements | "不含 main 写入 / 不含 push / 不含 PR / 不含 merge" |
| one-shot token hint | 一次性 token 提示 |
| expiry time | token 过期时间 |

### 8.3 确认流程

```text
1. System → 展示 PilotRequest（含所有 §8.2 信息）
2. User → 审查所有信息
3. User → 点击 "我确认此次试点写入"（明确措辞，不含"approve all"语义）
4. System → 生成 OneShotApprovalToken
5. System → 执行器在 token 有效期内执行写入
6. System → token 标记为 consumed
7. System → 记录 audit event
```
