### 5.3 API 行为边界

| 行为 | 是否执行 |
|------|---------|
| 生成 raw command / args / env | **否** — DTO 不含此类字段，validator 拒绝疑似命令文本 |
| 发 token / consume token | **否** — 无 approve / confirm / consume / token endpoint |
| 调用 ControlledRuntimeService | **否** — readback builder 只依赖 preflight/preview/disabled adapter |
| 调用 ExecutorService | **否** |
| 调用 Worker | **否** |
| 读取环境变量 / 密钥 | **否** — 所有 safe text validator 拒绝疑似 secret 文本 |
| 探测本机 CLI | **否** |
| 使用 subprocess / shell / os.popen / pty / terminal / tmux | **否** |

### 5.4 ReadbackBuilder 组装链路

```
RealExecutorLaunchReadbackRequest
  → to_launch_context() → RealExecutorLaunchContext
  → RealExecutorPreflightService.evaluate_launch(context)
      → RealExecutorPreflightResult (ready/blocked + safety boundary blocking reasons)
  → RealExecutorLaunchPlanPreviewBuilder.build(RealExecutorLaunchPlanPreviewInput)
      → RealExecutorLaunchPlanPreview (ready/blocked + display steps)
  → DisabledRealExecutorAdapter.launch(context)
      → RealExecutorOperationResult (blocked + real_executor_disabled reason)
  → RealExecutorLaunchReadbackResponse
      adapter_enabled=false, adapter_launch_status=blocked,
      preview_executable=false, real_executor_launch_started=false,
      product_runtime_git_write_allowed=false, api_mode="read_only"
```

## 9. 风险与下一步

### 9.1 当前风险

| 风险 | 说明 |
|------|------|
| **真实执行器仍未启动** | 当前所有 real executor 能力均被 disabled adapter 拦截；真实 CLI 启动、进程管理、stdout/stderr capture、signal kill 均未实现 |
| **无真实进程管理** | 没有 process_id 追踪、无 heartbeat 检测、无 exit_code 捕获 |
| **无真实 activity detection** | 无法从原生 JSONL / terminal output 检测执行器活动状态 |
| **无真实 cost/token readback** | 无法从原生 session file 回填 token usage |
| **readback 链路未经人工核验** | readback API 和 frontend panel 已有测试覆盖但未经用户在其实环境中手动核验 |

### 9.2 下一步规则

1. **下一步不能直接启动真实执行器。**
2. **下一步应由用户确认是否进入 P9-REL-H：manual UI/backend readback validation。**
3. **P9-REL-H 仍是人工验证 readback，不是启动真实执行器。**
4. **真正进入 P9-REL-Pilot 前必须再做一次用户显式确认。**

### 9.3 推荐下一步

| 推荐步骤 | 阶段 | 说明 |
|---------|------|------|
| **用户确认** | P9-REL-G Gate 确认 | 用户阅读本文档并确认 Pass / Ready for user manual validation |
| **P9-REL-H（推荐）** | Manual UI/backend readback validation | 用户手动调用 `POST /runtime/real-executor/launch-readback`，在前端查看 readback card 展示，确认 response 字段与文档一致 |
| **P9-REL-Pilot（需用户显式确认）** | 真实执行器启动试点 | 仅用户未来单独确认后执行，不在此阶段自动推进 |
