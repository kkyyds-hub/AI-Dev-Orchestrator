## 2. P8 范围定义

### 2.1 P8 是什么

P8 只做 **executor config discovery / registry contract / readiness readback / launch preview design**。

P8 回答的问题：
- 系统里有哪些可用的外部执行器？
- 每个执行器的能力边界是什么？
- 如何只读发现本地 CLI 安装状态和原生配置？
- 如何生成 launch command preview 而不执行？
- Settings 页面如何展示执行器就绪状态？

### 2.2 P8 做什么

| # | 做什么 | 说明 |
|---|--------|------|
| 1 | ExecutorProfile 域模型 | executor_id / display_name / capabilities / status |
| 2 | ExecutorConfigDiscovery 域模型 | CLI 安装状态 / 版本 / 登录状态 / 配置来源 |
| 3 | ExecutorRegistry 域模型 | 管理所有已注册 ExecutorProfile |
| 4 | 安全只读发现 | metadata only，allowlist，no secret |
| 5 | ExecutorLaunchPreview 域模型 | 命令预览 / 就绪状态 / 阻止原因 |
| 6 | 只读 API | 前端 settings/workbench readback（只显示状态，不启动） |
| 7 | Launch preview 设计 | 只生成 preview，不执行 |

### 2.3 P8 不做（硬边界）

| # | 不做 | 原因 |
|---|------|------|
| 1 | 真实启动外部执行器（codex / claude 进程） | 属于 P9 |
| 2 | 执行任何 CLI 命令 | P8 只做 preview |
| 3 | 写入原生配置文件 | 只读发现 |
| 4 | 自动安装执行器 | 用户自行管理 |
| 5 | 实现 executor session / run state | 属于 P9 |
| 6 | terminal log capture | 属于 P9 |
| 7 | activity detection | 属于 P9 |
| 8 | cost / token usage 回填 | 属于 P9 |
| 9 | 读取 ~/.codex、~/.claude、环境变量真实值、API key、token、base_url 明文 | 安全边界 |
| 10 | 读取用户真实本机配置 | 安全边界 |

## 4. Executor Config Discovery 安全模型

### 4.1 核心安全原则

| # | 原则 | 规则 |
|---|------|------|
| 1 | **不记录 secret** | 不记录 secret / token / API key 明文 |
| 2 | **不 dump 环境变量** | 不 dump 环境变量 |
| 3 | **不 dump 原生配置文件** | 不 dump 原生配置文件全文 |
| 4 | **不读取用户真实本机配置** | 不读取用户真实本机配置 |
| 5 | **不展示敏感路径** | 不展示敏感路径，除非后续用户显式配置 allowed path |
| 6 | **只记录元数据** | 只记录"是否存在 / 是否可用 / 来源类型 / 状态 / 更新时间 / 错误摘要" |
| 7 | **discovery result 是只读 evidence** | 不等于可以启动 |

### 4.2 安全字段设计

#### 允许记录的字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `executor_id` | str | "codex" / "claude_code" / "deepseek" |
| `cli_installed` | bool | 二进制是否在 PATH 中 |
| `cli_version` | str \| None | 版本号（如 "0.2.0"） |
| `login_status` | str | "logged_in" / "not_logged_in" / "expired" / "unknown" |
| `default_model` | str \| None | 默认模型名（不包含 API key） |
| `permission_mode` | str \| None | 当前权限模式 |
| `config_source` | str | "native_config" / "env" / "project_managed" / "none" |
| `native_config_valid` | bool | 原生配置是否有效 |
| `token_configured` | bool | token 是否已配置（不记录 token 值） |
| `last_checked_at` | datetime \| None | 最后检查时间 |
| `discovery_error` | str \| None | 发现过程中的错误摘要（不含敏感信息） |

#### 禁止记录的字段

| # | 禁止字段 | 原因 |
|---|---------|------|
| 1 | API key 明文 | 安全红线 |
| 2 | auth token 明文 | 安全红线 |
| 3 | cookie / session token | 安全红线 |
| 4 | 原生配置文件完整内容 | 可能包含敏感信息 |
| 5 | 环境变量完整 dump | 可能包含 secret |
| 6 | 未脱敏的绝对路径（如 `~/.codex/config.toml`） | 除非在 allowed path 白名单中 |
| 7 | `env_vars_present` 中列出具体变量名 | 改用计数 `env_var_count`（但 P8 路线文档中的 `env_vars_present: list[str]` 需在实现时修订为安全版本） |

### 4.3 安全模型对比：ProviderConfigService 已有的 mask 模式

`ProviderConfigService` 已有成熟的安全模式可供 P8 复用：

| ProviderConfigService 模式 | P8 复用方式 |
|---------------------------|------------|
| `_mask_api_key()` → `********xxxx` | ExecutorConfigDiscovery.token_configured 只返回 bool，不返回任何 key 片段 |
| `source: ProviderConfigSource` → "saved_config" / "env" / "none" | ExecutorConfigDiscovery.config_source: "native_config" / "env" / "project_managed" / "none" |
| `detect_provider_type()` → 从 base URL 推断 provider | ExecutorConfigDiscovery 可类似推断 executor 类型，但不暴露完整 URL |
| `OpenAIProviderConfigSummary` → 安全摘要，不含 plaintext key | ExecutorConfigDiscovery 本身就是安全摘要 |

## 5. P8 子阶段拆分建议

### 5.1 子阶段总览

| 子阶段 | 内容 | 负责人建议 | 说明 |

| **P8-A** | ledger + current-state audit + safety boundary | DeepSeek | **本文档** |

| **P8-Final** | targeted evidence + P8 Gate | DeepSeek | P8 evidence 收口 |

### 5.2 各子阶段详细说明

#### P8-A（本文档）

- **输入**: P7 Final Gate Pass、P8/P9 路线文档
- **产出**: P8-A ledger、现状审计矩阵、安全模型、子阶段拆分、P8 Gate 草案
- **不产出**: 任何代码、任何 API、任何前端
- **Gate**: 本文档完成即 P8-A Pass

#### P8-B：ExecutorProfile / DiscoveryResult contract freeze

- **输入**: P8-A ledger、P8/P9 路线文档中的域模型草案
- **产出**:
  - `ExecutorProfile` 域模型冻结（executor_id / display_name / provider / binary_name / capabilities / permission_model / status）
  - `ExecutorCapability` 域模型冻结
  - `ExecutorConfigDiscovery` 域模型冻结（安全字段版本）
  - `ExecutorRegistry` 域模型冻结（纯 registry 接口，不实现）
  - `ExecutorLaunchPreview` 域模型冻结
  - `ExecutorLaunchSafetyFlags` 域模型冻结
- **不产出**: service 实现、API、CLI 调用、真实配置读取
- **Gate**: contract freeze 完成，无安全字段泄露

#### P8-C：safe discovery service 最小实现

- **输入**: P8-B contract freeze
- **产出**:
  - `ExecutorConfigDiscoveryService` 最小实现
  - 仅探测 CLI 二进制是否存在（`which codex` / `which claude`）
  - 仅探测版本号（`codex --version` / `claude --version`）
  - 仅探测登录状态（从 CLI 返回码推断，不读配置文件）
  - 所有探测结果通过 allowlist 过滤
  - 错误摘要脱敏
- **不产出**: 配置文件读取、环境变量读取、API key 检测、token 检测
- **Gate**: discovery service 返回 metadata only，无 secret exposure

#### P8-D：read-only API + settings/workbench readback

- **输入**: P8-C discovery service
- **产出**:
  - `GET /executors` — 列出所有已注册 executor profile
  - `GET /executors/{executor_id}` — 获取单个 executor profile + discovery status
  - `GET /executors/{executor_id}/readiness` — 获取 executor readiness readback
  - Settings 页面只读展示执行器状态卡片
  - Workbench 只读展示 executor 就绪状态
- **不产出**: 启动按钮、测试连接按钮、配置编辑表单
- **Gate**: API 只读，不触发任何外部进程或文件写入

#### P8-E：launch preview contract

- **输入**: P8-C discovery service + P8-D API
- **产出**:
  - `ExecutorLaunchPreview` 生成逻辑
  - `GET /executors/{executor_id}/launch-preview` — 返回 launch preview（命令字符串、就绪状态、阻止原因）
  - `POST /executors/{executor_id}/launch-preview` — 针对特定 task/run 生成 preview
  - workspace 绑定验证（复用 P4 workspace 逻辑）
  - 安全 flag 生成
- **不产出**: 实际启动、subprocess 调用、executor session 创建
- **Gate**: launch preview 只生成命令预览，不执行

#### P8-Final：targeted evidence + P8 Gate

- **输入**: P8-B 至 P8-E 所有产出
- **产出**:
  - P8 阶段 targeted evidence 汇总
  - P8 Gate 检查清单验证
  - P8 Final Gate 结论
- **Gate**: P8 Final Gate Pass 或 Blocked

### 5.3 P8 与 P9 边界

| 维度 | P8（本阶段） | P9（下一阶段） |
|------|------------|------------|
| Executor 生命周期 | 只管理 Profile/Registry（静态配置视图） | 管理 ExecutorSession/ExecutorRunState（运行时状态机） |
| 进程 | 不启动任何外部进程 | 受控启动 codex/claude 进程 |
| 命令执行 | 只生成 launch_command_preview 字符串 | 实际执行 CLI 命令 |
