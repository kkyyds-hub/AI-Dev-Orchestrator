# V5 里程碑裁定线程检查清单

## 开始前

- 已明确当前对象是工作包 / 单轮切片 / Phase 哪一层。
- 已明确这次只裁定什么，不裁定什么。
- 已确认当前主任务真的是 accept，而不是 build / write / govern / drive / verify。

## 材料核对

- 已看到母本或 freeze 中对应的完成定义。
- 已看到本轮实现说明或改动摘要。
- 已看到最近一轮 review 结论。
- 已看到最近一轮 verify 结论。
- 若材料不足，已记录具体缺证项。

## 结论判断

- 已区分当前是 `Pass / Partial / Blocked / 无法裁定` 的哪一种。
- 已明确证据支撑的范围，不夸大到更大范围。
- 已明确当前是缺证，还是被真实事实阻塞。
- 若对象涉及结构治理，已检查 build / 入口 / 回归 / 锚点影响是否有事实支撑。

## 正式输出

- 已写明能确认完成的范围。
- 已写明不能确认完成的范围。
- 已写明风险、缺证项、必补项与下一阶段前置条件。
- 已指定下一线程 owner skill。
- 已避免把 partial 写成 pass，把无法裁定写成 blocked。
