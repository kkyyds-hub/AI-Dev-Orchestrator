# V5 Pass / Partial / Blocked 规则

## 1. 目的

统一 `accept-v5-milestone-gate` 的裁定表达，避免因为表达模糊导致后续线程误解状态。

## 2. Pass

优先适用于：

- 范围明确且闭合
- 关键实现已经完成
- 最小必要验证通过
- 重大风险已消化或已被接受
- 文档、代码、verify 口径一致

### Pass 结论里必须写

- 到底什么范围 pass 了
- 不是哪些更大范围也一起 pass 了

## 3. Partial

优先适用于：

- 核心部分完成，但有明确残留缺口
- 切片完成，但阶段未完成
- 可以继续往下推进，但必须保留未完成项和前置条件

### 最典型的 Partial 场景

- backend OK，web 未闭合
- web 有入口，联调证据不足
- API 存在，但主链未完全接上
- verify 只覆盖最小切片，尚不足以支撑阶段级 pass

## 4. Blocked

优先适用于：

- 关键实现未完成
- 缺少最小验证
- 高风险未澄清
- 口径明显不一致，当前无法可靠裁定

### Blocked 结论里必须写

- 被什么阻塞
- 先补什么
- 先由哪个 skill 接手

## 5. 一个基本纪律

当你犹豫是 `Pass` 还是 `Partial` 时，默认先给 `Partial`。

因为 V5 的验收目标是诚实收口，不是乐观表态。
