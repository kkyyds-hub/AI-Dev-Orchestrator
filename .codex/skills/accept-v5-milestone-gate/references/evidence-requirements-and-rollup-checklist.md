# V5 验收证据与汇总检查清单

## 1. 目的

很多错误验收不是因为不会判断，而是因为材料不齐。

这个清单用于快速判断：当前材料够不够支撑一个可靠裁定。

## 2. 最低材料要求

理想情况下，至少同时具备：

- 完成定义或计划口径
- 实现摘要
- review 结论
- verify 结论

如果缺其中任一项，要明确写：

- 缺了什么
- 这会限制你把结论说到哪一步

## 3. 工作包级裁定最低要求

优先至少确认：

- 该工作包的目标是什么
- 当前实现覆盖了哪些目标
- 当前 verify 覆盖了哪些目标
- 当前 review 有没有高风险阻断

## 4. Phase 级裁定额外要求

除了工作包级材料，还要确认：

- Phase 下关键工作包是否基本闭合
- 遗留项是否仍可能推翻阶段结论
- 是否已有足够 build / smoke / page / API 证据支撑更大范围结论

## 5. 遗留项检查

每次裁定至少留下：

- 风险
- 必补项
- 下一阶段前置条件
- 下一线程建议 skill

没有这些信息的裁定，不适合做接力基线。

## 6. 推荐交接规则

- 缺完成定义 → `manage-v5-plan-and-freeze-docs`
- 缺实现 → `write-v5-runtime-backend` / `write-v5-web-control-surface`
- 缺风险澄清 → `review-v5-code-and-risk`
- 缺运行证据 → `verify-v5-runtime-and-regression`
- 本质跨层 → `drive-v5-orchestrator-delivery`

## 7. 一句话纪律

验收不是为了“宣布成功”，而是为了把当前真实边界说清楚。
