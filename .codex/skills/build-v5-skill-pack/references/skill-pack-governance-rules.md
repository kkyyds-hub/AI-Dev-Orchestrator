# V5 skill pack 治理规则

## 1. 目的

把 `build-v5-skill-pack` 绑定到当前 V5 技能库的真实治理目标，避免技能体系越写越散。

这个参考文件回答三个问题：

1. 什么样的内容应该进入 V5 正式技能库
2. 什么样的内容不应该单独变成 skill
3. 技能库应该怎样持续保持风格与边界一致

## 2. 什么应该进入正式技能库

优先进入：

- 有明确 owner 边界的 workflow skill
- 能指导后续线程直接接手的 skill
- 明确绑定 V5 Phase / 工作包 / 验证 / 验收口径的 skill
- 已准备正式落到 `.codex/skills/` 的 skill

## 3. 什么不应该单独变成 skill

不优先进入：

- 只有能力主题、没有 owner 边界的名字
- 纯模块分类、纯架构概念、纯能力清单
- 不准备正式落盘的临时脑暴材料
- 与已有 skills 明显重叠、却没有新的独立交接价值的内容

## 4. 风格统一原则

当前 V5 正式 skills 已逐步形成统一风格：

- 中文详细版
- 明确 owner
- 明确边界
- 明确开始入口
- 明确工作流
- 明确兄弟 skill 协作契约
- 明确 done checklist

新 skill 应继续延续，而不是另起一套写法。

## 5. 一句话纪律

V5 skill pack 追求的是“可接力的 owner 体系”，不是“看起来很多的能力列表”。
