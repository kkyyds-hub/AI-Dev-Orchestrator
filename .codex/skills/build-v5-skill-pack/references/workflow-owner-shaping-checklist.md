# workflow owner 化检查清单

## 1. 目的

把能力主题写成真正 skill，最难的不是补字数，而是把它写成一个可执行的 owner。

本清单用于快速判断：当前草案是否已经具备 workflow owner 资格。

## 2. 先问三个问题

### 这个 skill 管什么？

如果一句话答不出来，就还不是 owner。

### 这个 skill 不管什么？

如果边界说不清，就会和兄弟 skill 打架。

### 线程结束时它应该留下什么？

如果没有明确交接物，就不适合做 workflow skill。

## 3. 必须出现的要素

至少应有：

- 使命与 owner
- 强绑定输入
- 技能边界
- 开始入口
- 核心工作流
- 兄弟 skill 协作契约
- done checklist

## 4. 常见失败写法

- 只描述能力，不描述 owner
- 只描述主题，不描述触发条件
- 只描述理想目标，不描述线程结束要留下什么
- 只讲大词，不讲目录、文件、证据、接棒关系

## 5. 一句话纪律

能直接指导一个新线程接手，才算完成了 owner 化。
